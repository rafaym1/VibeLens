# Multi-Channel Installation Design

## Goal

Make VibeLens installable via multiple methods for both technical and non-technical users across macOS, Linux, and Windows.

## Install Methods

| Method | Command | Audience |
|--------|---------|----------|
| pip | `pip install vibelens` | Python users |
| npx | `npx vibelens` | Node.js users (requires Python on system) |
| Developer | `uv pip install -e ".[dev]"` | Contributors |

PyPI is the single source of truth. npm is a distribution wrapper that installs the same version from PyPI.

**Homebrew dropped:** VibeLens's dependency tree contains 15+ Rust-based packages (pydantic-core, jiter, tiktoken, aiohttp, etc.) that Homebrew cannot build from source. Revisit when Homebrew improves binary wheel support or deps simplify.

## Channel 1: pip (existing)

Already works via `pip install vibelens`. No changes needed to the package itself. Polish the README install section to list all methods clearly.

## Channel 2: npm Package

**Package name:** `vibelens` on npm. Version numbers match PyPI exactly (e.g., npm `0.9.31` = PyPI `0.9.31`).

**Structure:**
```
npm/
├── package.json
├── bin/
│   └── vibelens.js
└── README.md
```

**How `bin/vibelens.js` works (~80 lines):**

1. Detect Python on the system:
   - Try `python3`, then `python`, then `py -3` (Windows launcher)
   - Support `VIBELENS_PYTHON` env var for non-standard paths
   - Verify version >= 3.10
2. Check if vibelens is installed: `python -m vibelens version`
3. If installed: `spawnSync(python, ["-m", "vibelens", ...process.argv.slice(2)])`
4. If Python found but vibelens missing: print install instructions (`pip install vibelens`)
5. If no Python found: print platform-specific Python install guidance

**No `postinstall` script.** All detection at runtime. Works with `--ignore-scripts`.

## Channel 3: Homebrew Tap

**Tap repo:** `CHATS-lab/homebrew-tap` on GitHub.

**Install:**
```bash
brew install chats-lab/tap/vibelens
```

**Formula:** Uses `Language::Python::Virtualenv` with resource blocks generated by `homebrew-pypi-poet`. Depends on Homebrew's `python@3.12`. Creates isolated virtualenv in `libexec/`.

**Pre-built bottles:** GitHub Actions builds bottles on macOS arm64, macOS x64, and Linux x64 on each release. Users download pre-compiled archives for near-instant install.

**Tap repo structure:**
```
homebrew-tap/
├── Formula/
│   └── vibelens.rb
├── .github/
│   └── workflows/
│       └── update-formula.yml
└── README.md
```

**Future:** Once the project has sufficient traction (30+ stars/forks), submit to homebrew-core for `brew install vibelens`.

## Channel 4: Developer Install

```bash
git clone https://github.com/CHATS-lab/VibeLens && cd VibeLens
uv pip install -e ".[dev]"
```

No Makefile, no scripts. `uv` handles venv creation and editable install.

## Release Workflow

On each new version:

1. Version bump (`pyproject.toml` + `__init__.py`)
2. Changelog entry
3. Commit, push, tag, GitHub Release
4. PyPI upload (`python -m build && twine upload dist/*`)
5. npm publish (automated via CI or manual `cd npm && npm publish`)
6. Homebrew formula update (main repo release workflow triggers tap repo's `update-formula.yml` via `gh workflow run`)

### Homebrew CI automation

`update-formula.yml` in the tap repo:
1. Installs new vibelens version + `homebrew-pypi-poet` in temp venv
2. Runs `poet -f vibelens` to regenerate formula with all resource blocks and SHA256s
3. Patches `desc`, `homepage`, `license`, `test` block
4. Builds bottles on 3 runners (macOS arm64, macOS x64, Linux x64)
5. Uploads bottle archives to GitHub Release
6. Patches `bottle do` block with SHA256s
7. Commits and pushes

### npm CI automation

On GitHub Release, a workflow in the main repo:
1. Updates `npm/package.json` version to match the release tag
2. Runs `npm publish` from the `npm/` directory

## Decisions

- **No bundled `uv` in npm:** Adds too much complexity for marginal benefit. Users who have Node likely can install Python.
- **No `curl | sh` installer:** Less discoverable than npm, trust barrier for some users.
- **No homebrew-core submission yet:** Project needs more traction first. Tap with bottles provides equivalent UX.
- **Full install only:** No lite/core-only mode. All install methods get all features.
- **All platforms:** macOS, Linux, Windows supported across pip and npm. Homebrew covers macOS + Linux.
